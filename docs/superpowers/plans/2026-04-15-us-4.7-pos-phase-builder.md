# US-4.7 POS Phase-Builder Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give the office operator a first-class phase-builder for POS (Piano Operativo di Sicurezza) — editable phase entities with per-phase risks / NIOSH / noise / vibration, drag-and-drop reordering, dependency linking, and a Gantt-like overview in the generated .docx.

**Architecture:** Keep phases embedded as a JSON list on the existing `pos` row (no separate table — current docx generator already consumes the list). Give every phase a stable `id` (UUID string) so reorders + dependencies survive edits. Add a sibling `phase_dependencies` JSONB column for the edge list. Frontend gets a new `<PosPhaseBuilder>` section in the existing `/assessments/pos/[aziendaId]` page using `@dnd-kit` + `react-hook-form` `useFieldArray` (same pattern as MMC `mmc-form.tsx`). Docx generator gains a per-phase detail block + Gantt overview section.

**Tech Stack:**
- Backend: SQLAlchemy 2 async + Pydantic v2 + Alembic + FastAPI + python-docx + pytest.
- Frontend: Next.js 16 (App Router — see `node_modules/next/dist/docs/` before editing), react-hook-form + zod + `@dnd-kit/core` + `@dnd-kit/sortable` + shadcn/ui + Framer Motion.

**Acceptance Criteria (US-4.7):**
- **AC1**: Adding a phase lets me attach phase-specific risks + NIOSH + noise/vibration.
- **AC2**: Drag-and-drop order persists and is reflected in the generated .docx.
- **AC3**: Phase dependency links are shown both in the UI and in the .docx Gantt-like overview.

```python
class PhaseRisk(BaseModel):
    """Single risk entry attached to a phase.

    P/D/I mirror the DVR scale. P and D are 1-4 integers; I = 2*D + P is
    derived on the frontend and resent for audit. We keep `categoria`
    free-form so the frontend can add new labels without backend churn.
    """

    categoria: str = Field(..., min_length=1, max_length=120)
    descrizione: str | None = Field(None, max_length=1000)
    p: int = Field(..., ge=1, le=4, alias="P")
    d: int = Field(..., ge=1, le=4, alias="D")
    i: int = Field(..., ge=3, le=12, alias="I")

    model_config = ConfigDict(populate_by_name=True)


class PhaseNiosh(BaseModel):
    """Per-phase NIOSH summary.

    The operator runs the full MMC worksheet elsewhere and drops the
    worst-lift summary here. `zona` mirrors VERDE/GIALLA/ROSSA so the
    docx can highlight without re-computing.
    """

    peso_reale_kg: float = Field(..., gt=0)
    cp: float = Field(..., gt=0)
    ir: float = Field(..., ge=0)
    zona: str = Field(..., pattern=r"^(VERDE|GIALLA|ROSSA)$")
    note: str | None = Field(None, max_length=500)


class PhaseRumore(BaseModel):
    lex_8h_dba: float = Field(..., ge=0, le=140)
    fascia: str = Field(..., max_length=30)  # e.g. "< 80", "80-85", "85-87", "> 87"
    dpi_obbligatori: bool = False
    note: str | None = Field(None, max_length=500)


class PhaseVibrazioni(BaseModel):
    a8_mano_braccio: float = Field(..., ge=0, le=50)
    a8_corpo_intero: float = Field(..., ge=0, le=50)
    entro_limiti: bool = True
    note: str | None = Field(None, max_length=500)


class Phase(BaseModel):
    """One phase entity inside pos.fasi_lavorative.

    `id` is assigned by the backend on first save so drag/drop order
    and dependency edges survive renames.
    """

    id: str | None = Field(None, description="UUID assigned by backend")
    fase: str = Field(..., min_length=1, max_length=120)
    descrizione: str | None = Field(None, max_length=2000)
    data_inizio: date | None = None
    data_fine: date | None = None
    rischi: list[PhaseRisk] = Field(default_factory=list)

    mezzi: list[str] = Field(default_factory=list)
    niosh: PhaseNiosh | None = None
    rumore: PhaseRumore | None = None
    vibrazioni: PhaseVibrazioni | None = None


class PhaseDependency(BaseModel):
    """Directed edge linking two phases in the same POS."""

    from_id: str
    to_id: str
    type: str = Field("fs", pattern=r"^(fs|ss|ff)$")
    lag_days: int = Field(0, ge=-365, le=365)
    note: str | None = Field(None, max_length=300)

    Phases missing an `id` get one assigned. Edges whose `from_id` /
    `to_id` do not appear in `phases` are rejected with 422.

    phases: list[Phase]
    dependencies: list[PhaseDependency] = Field(default_factory=list)
```

```python
from app.schemas.pos import PhasesReplace  # add to imports


@router.put("/{pos_id}/phases", response_model=PosResponse)
async def replace_phases(
    azienda_id: uuid.UUID,
    pos_id: uuid.UUID,
    body: PhasesReplace,
    db: AsyncSession = Depends(get_db),
):
    """Atomic replace of the ordered phase list + dependency edges.

    Backend assigns a UUID to any phase missing an `id` so the edge list
    stays valid across reorders. Any dependency edge that references an
    unknown phase id returns 422.
    """
    pos = await _get_pos_or_404(db, azienda_id, pos_id)

    # Assign IDs to fresh phases, preserve existing IDs verbatim.
    seen: set[str] = set()
    new_phases: list[dict] = []
    for p in body.phases:
        pid = p.id or str(uuid.uuid4())
        if pid in seen:
            raise HTTPException(status_code=422, detail=f"Duplicate phase id {pid}")
        seen.add(pid)
        new_phases.append({**p.model_dump(exclude_none=False), "id": pid})

    # Validate dependency edges point at known phase IDs.
    for e in body.dependencies:
        if e.from_id not in seen or e.to_id not in seen:
            raise HTTPException(
                status_code=422,
                detail=f"Dependency references unknown phase id ({e.from_id} -> {e.to_id})",
            )
        if e.from_id == e.to_id:
            raise HTTPException(status_code=422, detail="Phase cannot depend on itself")

    pos.fasi_lavorative = new_phases
    pos.phase_dependencies = [e.model_dump() for e in body.dependencies]
    await db.commit()
    await db.refresh(pos)
    return pos
```

```python
def _render_phase_overview(doc, pos) -> None:
    """US-4.7 AC3: emit a dependency-aware overview of the phase plan.

    We render TWO things because a real Gantt chart in .docx would need
    a drawing shape surface that python-docx only exposes half of:

      1. Phase schedule table (ordered rows with `fase`, `data_inizio`,
         `data_fine`, and an ASCII bar sized by `TOTAL_SLOTS`).
      2. Dependency list (`Predecessore -> Successore  (Tipo, lag X g.)`)
         so the reader can see the dependency network even when the
         bars aren't drawn.
    """
    fasi = pos.fasi_lavorative or []
    deps = pos.phase_dependencies or []
    if not fasi:
        return
    add_heading(doc, "Panoramica fasi (Gantt)", level=3)

    from datetime import date as _date, datetime as _datetime

    def _parse(d):
        if not d:
            return None
        if isinstance(d, (_date, _datetime)):
            return d if isinstance(d, _date) else d.date()
        try:
            return _datetime.strptime(str(d)[:10], "%Y-%m-%d").date()
        except ValueError:
            return None

    starts = [_parse(f.get("data_inizio")) for f in fasi]
    ends = [_parse(f.get("data_fine")) for f in fasi]
    valid = [s for s in starts if s] + [e for e in ends if e]
    TOTAL_SLOTS = 30

    def _bar(s, e):
        if not s or not e or not valid:
            return "—"
        span_start = min(valid)
        span_end = max(valid)
        total_days = max((span_end - span_start).days, 1)
        pre = int(round(((s - span_start).days / total_days) * TOTAL_SLOTS))
        dur = max(int(round(((e - s).days / total_days) * TOTAL_SLOTS)), 1)
        return " " * pre + "█" * dur

    rows = []
    for f, s, e in zip(fasi, starts, ends):
        rows.append([
            f.get("fase", ""),
            s.strftime("%d/%m/%Y") if s else "—",
            e.strftime("%d/%m/%Y") if e else "—",
            _bar(s, e),
        ])
    add_data_table(doc, ["Fase", "Inizio", "Fine", "Durata"], rows)

    if deps:
        add_paragraph(doc, "Dipendenze:", bold=True)
        by_id = {f.get("id"): f.get("fase", "") for f in fasi}
        dep_rows = []
        for e in deps:
            dep_rows.append([
                by_id.get(e.get("from_id"), "?"),
                by_id.get(e.get("to_id"), "?"),
                (e.get("type") or "fs").upper(),
                f"{e.get('lag_days', 0)} g",
            ])
        add_data_table(doc, ["Predecessore", "Successore", "Tipo", "Lag"], dep_rows)
```

- [ ] **Step 1: Build the top-level component**

Create `frontend/src/components/assessments/pos/phase-builder.tsx`:

```tsx
"use client";

import { useCallback, useEffect, useMemo, useState } from "react";
import { useFieldArray, useForm } from "react-hook-form";
import { zodResolver } from "@hookform/resolvers/zod";
import {
  DndContext,
  KeyboardSensor,
  PointerSensor,
  closestCenter,
  useSensor,
  useSensors,
  type DragEndEvent,
} from "@dnd-kit/core";
import {
  SortableContext,
  sortableKeyboardCoordinates,
  verticalListSortingStrategy,
} from "@dnd-kit/sortable";
import { Plus, Save } from "lucide-react";
import { toast } from "sonner";

import { Button } from "@/components/ui/button";
import { Card, CardContent, CardHeader, CardTitle } from "@/components/ui/card";
import { useApi } from "@/hooks/use-api";

import { PhaseCard } from "./phase-card";
import { PhaseDependencies } from "./phase-dependencies";
import {
  DEFAULT_PHASE,
  phaseBuilderSchema,
  type PhaseBuilderValues,
  type Phase,
  type PhaseDependency,
} from "./phase-schema";

interface PhaseBuilderProps {
  aziendaId: string;
  posId: string;
  initialPhases: Phase[];
  initialDependencies: PhaseDependency[];
  onSaved?: (v: PhaseBuilderValues) => void;
}

function genLocalId(): string {
  return `local-${Math.random().toString(36).slice(2, 10)}`;
}

export function PhaseBuilder({
  aziendaId,
  posId,
  initialPhases,
  initialDependencies,
  onSaved,
}: PhaseBuilderProps) {
  const { apiFetch } = useApi();
  const [saving, setSaving] = useState(false);

  // Ensure every phase has a stable key for DndKit even before the backend
  // has minted its UUID — we replace the local key with the real id once
  // the PUT response lands.
  const form = useForm<PhaseBuilderValues>({
    resolver: zodResolver(phaseBuilderSchema),
    defaultValues: {
      phases: initialPhases.map((p) => ({
        ...DEFAULT_PHASE,
        ...p,
        id: p.id ?? genLocalId(),
      })),
      dependencies: initialDependencies,
    },
    mode: "onBlur",
  });

  const phasesArray = useFieldArray({
    control: form.control,
    name: "phases",
    keyName: "_rhfId",
  });

  const phases = form.watch("phases");

  const sensors = useSensors(
    useSensor(PointerSensor, { activationConstraint: { distance: 5 } }),
    useSensor(KeyboardSensor, { coordinateGetter: sortableKeyboardCoordinates }),
  );

  const handleDragEnd = useCallback(
    (ev: DragEndEvent) => {
      const { active, over } = ev;
      if (!over || active.id === over.id) return;
      const oldIdx = phases.findIndex((p) => p.id === active.id);
      const newIdx = phases.findIndex((p) => p.id === over.id);
      if (oldIdx < 0 || newIdx < 0) return;
      phasesArray.move(oldIdx, newIdx);
    },
    [phases, phasesArray],
  );

  const sortableIds = useMemo(() => phases.map((p) => p.id as string), [phases]);

  const handleRemovePhase = useCallback(
    (idx: number) => {
      const removedId = phases[idx]?.id;
      phasesArray.remove(idx);
      // Strip dependency edges that touched the removed phase.
      if (removedId) {
        const remaining = (form.getValues("dependencies") ?? []).filter(
          (e) => e.from_id !== removedId && e.to_id !== removedId,
        );
        form.setValue("dependencies", remaining, { shouldDirty: true });
      }
    },
    [form, phases, phasesArray],
  );

  const onSave = form.handleSubmit(async (v) => {
    setSaving(true);
    try {
      // Backend accepts `id: null` for new phases; strip our local-* sentinels.
      const payload = {
        phases: v.phases.map((p) => ({
          ...p,
          id: p.id && !p.id.startsWith("local-") ? p.id : null,
        })),
        dependencies: v.dependencies,
      };
      const saved = await apiFetch<{
        fasi_lavorative: Phase[];
        phase_dependencies: PhaseDependency[];
      }>(`/api/v1/aziende/${aziendaId}/pos/${posId}/phases`, {
        method: "PUT",
        body: JSON.stringify(payload),
      });
      form.reset({
        phases: saved.fasi_lavorative.map((p) => ({ ...DEFAULT_PHASE, ...p })),
        dependencies: saved.phase_dependencies ?? [],
      });
      toast.success("Fasi salvate");
      onSaved?.({
        phases: saved.fasi_lavorative as Phase[],
        dependencies: saved.phase_dependencies ?? [],
      });
    } catch (err) {
      toast.error((err as Error).message || "Salvataggio fallito");
    } finally {
      setSaving(false);
    }
  });

  // Any dangling dependency edges the user created client-side but never
  // saved should be cleaned up on first mount so the dropdowns don't show
  // stale options.
  useEffect(() => {
    const ids = new Set(phases.map((p) => p.id));
    const cleaned = (form.getValues("dependencies") ?? []).filter(
      (e) => ids.has(e.from_id) && ids.has(e.to_id),
    );
    if (cleaned.length !== (form.getValues("dependencies") ?? []).length) {
      form.setValue("dependencies", cleaned);
    }
    // eslint-disable-next-line react-hooks/exhaustive-deps
  }, []);

  const isDirty = form.formState.isDirty;

  return (
    <form onSubmit={onSave} className="space-y-4">
      <Card>
        <CardHeader className="flex flex-row items-center justify-between">
          <CardTitle className="text-base">Fasi lavorative</CardTitle>
          <div className="flex gap-2">
            <Button
              type="button"
              variant="outline"
              size="sm"
              onClick={() =>
                phasesArray.append({ ...DEFAULT_PHASE, id: genLocalId() })
              }
            >
              <Plus className="mr-1 h-3 w-3" /> Aggiungi fase
            </Button>
            <Button type="submit" size="sm" disabled={saving || !isDirty}>
              <Save className="mr-1 h-3 w-3" />
              {saving ? "Salvataggio…" : "Salva fasi"}
            </Button>
          </div>
        </CardHeader>
        <CardContent className="space-y-2">
          {phasesArray.fields.length === 0 && (
            <p className="text-sm text-muted-foreground">
              Nessuna fase. Aggiungi la prima lavorazione del cantiere (es. Scavo).
            </p>
          )}
          <DndContext
            sensors={sensors}
            collisionDetection={closestCenter}
            onDragEnd={handleDragEnd}
          >
            <SortableContext items={sortableIds} strategy={verticalListSortingStrategy}>
              <div className="space-y-2">
                {phasesArray.fields.map((f, i) => (
                  <PhaseCard
                    key={f._rhfId}
                    id={(phases[i]?.id as string) || f._rhfId}
                    index={i}
                    form={form}
                    onRemove={() => handleRemovePhase(i)}
                  />
                ))}
              </div>
            </SortableContext>
          </DndContext>
        </CardContent>
      </Card>

      <PhaseDependencies form={form} />
    </form>
  );
}
```
